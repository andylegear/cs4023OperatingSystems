❓ Why must you close unused pipe ends?
A pipe EOF (read() returns 0) only arrives when all write-end file descriptors are closed. If the parent forgets to close its own copy of the write end, the child's read() blocks forever — the kernel thinks there is still a writer.
Pipe vs Shared Memory throughput:
Pipe~5 GB/s2 copies (user→kernel→user)
Shared mem~50 GB/s0 copies — DRAM speed
