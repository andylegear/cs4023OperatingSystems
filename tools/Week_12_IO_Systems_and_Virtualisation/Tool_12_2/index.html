What is the software stack? Every computing environment — bare-metal, VM, or container — can be described as a vertical stack of software layers, each depending on the one below. Understanding which layer exists (or is absent) in each model is the key to understanding isolation, overhead, and flexibility.
How to use this panel: Drag layer tiles from the bank into the correct numbered slots for each stack. Slot 1 = bottom (hardware); Slot 4 = top (application). When you're done, click Check to validate. Click Show Comparison to see the property table.
What are Linux namespaces? A namespace wraps a global OS resource so that each process inside a namespace sees its own isolated instance. Containers get a private set of namespaces — they think they own the machine, but they only own their namespace view.
How to use this panel: Click a namespace name on the left, then click the matching description on the right. Correct matches turn green. Try all 7!

VM or Container? The right choice depends on isolation requirements, startup time, OS compatibility, and resource overhead. VMs are needed when you require a different OS kernel, strong security isolation, or full OS features. Containers are preferred for microservices, fast startup, and lightweight packaging.
How to use this panel: For each deployment scenario, click VM, Container, or Either. Click Check All when done.
