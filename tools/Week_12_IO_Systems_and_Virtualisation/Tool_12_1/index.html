What is disk scheduling? A Hard Disk Drive (HDD) has a moving read/write head that must seek to the correct cylinder before reading data. Seek time dominates HDD latency (~2–15 ms). When multiple I/O requests are pending, the OS reorders them to minimise total head movement — this is disk scheduling.
How to use this panel: Choose an algorithm, set the head start position, and click Animate. Use Step → to advance one request at a time and observe the head marker moving. The movement counter shows cumulative cylinders travelled. The service order and algorithm notes appear on the right.

Select an algorithm and click Animate.
Algorithm Race: All four main disk scheduling algorithms run simultaneously on the same request queue. Compare total head movement side-by-side. The algorithm with the fewest cylinders is highlighted green; the worst is red.
FCFS serves requests in arrival order. SSTF always picks the closest. SCAN sweeps in one direction then reverses. C-SCAN sweeps outward only and wraps back to 0.
Custom experiment: Enter your own request queue and head position. Choose any algorithm and track size. Use this to verify your worked examples from the lecture or try Belady-style starvation scenarios.
Starvation demo: Try SSTF with requests 5, 6, 7, 8, 190 and head=5. Does cylinder 190 ever get served?
