Q4 — What is the bug?

/* Barrier — N threads must all arrive before any proceeds */
pthread_mutex_lock(&m);
arrived++;
if (arrived == N)
    pthread_cond_signal(&barrier_ready);  /* ← wake one */

What this tab tests: pthread_cond_signal() wakes exactly one waiting thread; pthread_cond_broadcast() wakes all waiting threads. Use signal when only one thread can act on the new state (e.g. one item becomes available for one worker). Use broadcast when all waiters can or must proceed (e.g. a barrier release, a lock becoming readable by multiple readers).
